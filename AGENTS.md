# AGENTS.md

## 1. 这个仓库是什么

这是一个面向长篇游戏剧本 / 叙事资料的通用 AI 工作台仓库。
它的目标不是只服务单个剧本，而是逐步建立一个可以处理：

- 普通文本检索
- 图谱检索 / 全局关系推理
- 主代理路由
- 未来聊天式创作协作

的长期项目底盘。

在做任何修改前，请先阅读：

- README.md
- PROJECT_SPEC.md
- docs/architecture.md

## 2. 当前阶段

当前阶段是 MVP Skeleton。

优先级：
1. 保持项目结构清晰
2. 先打通最小可运行闭环
3. 不要过度设计
4. 不要为了“看起来高级”而引入复杂方案

## 3. 开发原则

- 优先使用 Python
- 优先低依赖方案
- 优先可读性
- 优先小步迭代
- 优先让 GitHub / Python 新手看得懂
- 不要炫技
- 不要一次性引入太多抽象层
- 不要擅自把项目重构成复杂框架

## 4. 代码修改规则

- 在修改前，先阅读相关文件
- 优先做局部修改，不要无谓地改整个仓库
- 除非明确要求，否则不要删除已有文档内容
- 除非明确要求，否则不要引入数据库、部署配置或复杂前端
- 如果需求还不明确，请先补文档或 TODO，而不是假装系统已经完整实现
- 如果只是为了表达项目意图，优先更新 markdown 文档，而不是硬写占位代码

## 5. 文档要求

如果项目意图、模块职责或路线图还不清楚，优先补这些文档：

- README.md
- PROJECT_SPEC.md
- docs/architecture.md
- docs/roadmap.md

文档要真正能帮助未来协作者理解项目，而不是只写很短的摘要。

## 6. 实现偏好

当前实现偏好：
- txt / md 优先
- 本地文件优先
- 简单 loader / chunker 优先
- 纯 Python 数据结构优先
- 配置化优先于硬编码，但不要过度抽象

短期不追求：
- 生产级性能
- 完整 GraphRAG
- 花哨 UI
- 全套多 agent 编排

## 7. 每次任务完成时必须输出

请在任务结束时提供：

1. 改了哪些文件
2. 为什么这么改
3. 我下一步该检查什么
4. 当前还没做的部分是什么
5. 是否引入了任何新依赖

## 8. 如果任务复杂

如果任务复杂、模糊或涉及多个模块：
- 先使用 plan mode
- 先给出简短计划
- 再开始实现

## 9. 完成标准

一项修改算完成，至少要满足：
- 目标清楚
- 改动范围合理
- 文件结构没有被搞乱
- 文档与代码状态一致
- 新手能看懂这一步为什么存在
