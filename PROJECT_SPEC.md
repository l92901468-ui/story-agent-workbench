# PROJECT_SPEC.md

## 1. 项目愿景（Vision）

Story Agent Workbench 的长期目标是：

> 构建一个面向长篇游戏剧本 / 叙事资料的通用 AI 工作台，支持从“局部文本修改”到“全局叙事一致性分析”的完整创作过程。

它不是某个单一剧本项目的临时脚本集合，而是一个可持续扩展的基础设施仓库。

---

## 2. 这个项目要解决的问题

在长篇叙事创作中，常见痛点包括：

1. **资料体量巨大**：几十万字到上百万字，跨多个文档与版本。
2. **普通检索偏局部**：向量检索常能找到“相关段落”，但难以拼出“全局结构”。
3. **全局一致性难维护**：人物关系、阵营链条、时间线、伏笔回收、设定冲突常常跨文档出现。
4. **创作需求多样化**：不仅要“改一句话”，还要“看全局逻辑是否成立”。

本项目要同时支持：

- 局部文本层面的快速检索与改写辅助
- 全局关系层面的结构化分析与冲突提示

---

## 3. 目标用户

### Primary user
- 长篇剧情 / 游戏叙事创作者（个人或小团队）

### 当前典型协作者画像
- GitHub 新手
- Python 基础有限
- 希望仓库结构清晰、文档可读、可逐步维护

### 设计上的含义
- 代码与文档都要“新手可理解”
- 少依赖、少魔法、少隐藏流程
- 每一步都能解释“为什么存在”

---

## 4. 核心能力（Core Capabilities）

### 4.1 普通 RAG（文本检索）

用途：检索原文、草稿、设定、场景文本。

典型问题：
- “这段怎么改更顺？”
- “把某句原文找出来。”
- “这个 scene 还有哪些相关片段？”

### 4.2 GraphRAG（图谱检索 / 关系推理）

用途：处理跨文档全局关系。

关注对象：
- 人物关系网
- 阵营链条
- 时间线连接
- 伏笔回收网
- 隐含冲突

典型问题：
- “角色关系在全局上如何变化？”
- “哪些伏笔还没有回收？”
- “哪些设定互相冲突？”

### 4.3 主代理 / 路由器

用途：将用户问题路由到最合适的检索能力。

阶段策略：
- **当前**：仅做简单路由（文本 vs 图谱）
- **后续**：再扩展为多角色协同（如剧情陪聊、玩法建议、挑刺、构建器）

### 4.4 聊天式创作工作台（未来形态）

最终目标是聊天式协作体验，但当前不以 UI 为重点。

---

## 5. 数据分层与语义约定

### 5.1 数据分层

- **canon**：已确认、当前版本成立的内容（优先级高）
- **draft**：正在修改、未确认草稿（可被建议覆盖）
- **reference**：参考资料 / 灵感素材 / 外部模板（默认不当作事实）
- **structured registry**：结构化事实表（角色、地点、阵营、时间线锚点、别名等）

### 5.2 分层的意义

这不是文件夹命名习惯，而是系统行为边界：

- 检索排序依据
- 回答时的可信度来源说明
- 冲突检测时的优先级策略
- 未来版本对“事实更新流程”的基础约束

---

## 6. 当前阶段：做什么 / 不做什么

## 6.1 当前要做（MVP Skeleton）

1. 明确项目意图与边界（文档先行）
2. 保持 Python 包结构清晰
3. 打通最小可运行闭环：
   - 资料读取（基础）
   - 文本检索占位
   - 图谱检索占位
   - 主路由占位
4. 为后续实现留出稳定扩展点

### 6.2 当前不做

- 不实现完整 GraphRAG
- 不引入数据库与复杂部署
- 不做花哨前端
- 不一次性上复杂多 agent 编排
- 不追求生产级性能优化

---

## 7. 设计原则（Engineering Principles）

1. **Python first**
2. **Low dependency**
3. **Readable over clever**
4. **Small, verifiable loop first**
5. **文档与实现保持一致**
6. **避免过度抽象与过早优化**

---

## 8. 推荐目录职责（语义层面）

> 以下是职责定义，不要求在本轮任务中重构代码。

- `src/story_agent_workbench/main.py`：CLI 入口
- `src/story_agent_workbench/router/agent_router.py`：问题路由
- `src/story_agent_workbench/retrieval/text_retriever.py`：文本检索
- `src/story_agent_workbench/graph/graph_retriever.py`：图谱检索接口占位
- `src/story_agent_workbench/ingest/`：资料读取与预处理
- `src/story_agent_workbench/core/`：共享模型、配置、通用类型预留

---

## 9. 里程碑路线图（Roadmap）

### Milestone A：文档与骨架对齐（当前）
- 项目意图、术语、边界写清楚
- 最小入口与模块职责可读

### Milestone B：可验证最小检索闭环
- 本地文档读取
- 简单 chunk 与检索
- CLI 可演示基本查询

### Milestone C：结构化 registry 接入
- 角色/地点/阵营/别名/时间锚点表
- 可用于基础冲突提示

### Milestone D：GraphRAG 增量实现
- 关系抽取与简化图存储
- 跨文档问题的全局回答能力

### Milestone E：聊天式工作台
- 对话层接入
- 逐步支持创作协作角色

---

## 10. 完成标准（Definition of Done for current phase）

当前阶段一次有效改动至少满足：

1. 目标清楚（读者知道“这步在解决什么”）
2. 改动范围合理（局部、可验证）
3. 结构未被破坏（目录语义稳定）
4. 文档与代码状态一致
5. 新手可读可继续维护

---

## 11. 非目标声明（Non-goals, short-term）

以下内容是明确“暂不追求”：

- 全自动剧情生成系统
- 一步到位的企业级协作平台
- 复杂权限系统与在线多租户
- 完整生产运维体系

这些能力不是永远不做，而是当前阶段刻意延后。
